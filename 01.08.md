< 3. 객체지향 프로그래밍 >

* 객체의 정의 : 정보를 효율적으로 관리하기 위해 의미를 부여하고(ex 학생, 책) 분류하는 논리적 단위. 실제 세계에 존재하는 사물 또는 개념.
 - 자바의 8가지 타입 이외의 타입을(사용자 타입) 정의하여 사용하고 싶은 경우.
 - 구성요소 : 속성(필드)과 기능(메소드)

* 자바 타입 : byte / int / long / short / double / float / boolean / char


* 클래스의 정의 : 객체를 정의해 놓은 것.

* 인스턴스 : 객체가 메모리에 할당되어 실제 사용되는 경우

 6) API

< 필드 >
* 메모리 각 공간마다 1byte씩 할당
 - 각 변수 저장 영역은 4byte씩 할당됨(reference 변수도 4byte씩 할당됨)

* 객체 생성 -> heap에 저장(초기화)
  - stack에 값 저장, stack에 저장된 값은 참조 끝나면서 사라지고, heap에 저장된 값도 사라짐
  - static 변수는 class가 끝나기 전까지는 사라지지 않음

* 변수
 - 인스턴스 변수 : 인스턴스 마다 값 저장.
 - 클래스 변수 : 같은 클래스 내의 모든 인스턴스들이 공유하는 변수.
 - 지역 변수 : 메소드 내에 선언, 메소드의 종료와 함께 소멸됨.


* 메소드 호출
  - call (function) by value
  - call by reference

< 클래스- 접근자 >
 - public : 무조건 접근
 - protected : 상속에서 사용. 자식 -> 부모 접근을 위해 사용.
 - private : 하나의 클래스 내부

 - default(잘 사용은X. 위의 세가지를 다 붙이지 않은 경우) : 같은 패키지 내에서만 접근이 가능. 다른 패키지에서는 사용X
